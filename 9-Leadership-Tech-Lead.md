# Leadership & Tech Lead

## Question 201-225: Leadership Essentials

**Q201: Handling underperformers**
- Clear expectations
- Regular feedback (not surprises)
- Support: Training, mentoring
- Document issues
- PIP if needed
- Empathetic, focused on growth

**Q202: Mentoring strategy**
- 1-on-1 cadence (weekly)
- Career development plan
- Stretch assignments
- Knowledge sharing
- Lead by example
- Invest in growth

**Q203: Architecture governance**
- Architectural decision records (ADRs)
- Code review standards
- Tech stack decisions
- Scaling guidelines
- Documentation requirements

**Q204: Code review philosophy**
- Focus on: Logic, security, performance, maintainability
- NOT: Style (use linters)
- Be constructive, suggest improvements
- Approve good PRs quickly
- Explain "why" in feedback

**Q205: Managing technical debt**
- Track explicitly (backlog)
- Quantify impact
- Balance with features
- Refactor incrementally
- Prevent accumulation

**Q206: Stakeholder conflict**
- Understand all perspectives
- Find common ground
- Data-driven decisions
- Transparent communication
- Escalate if needed

**Q207: Delivery pressure**
- Don't compromise quality/security
- Push back with data
- Offer trade-offs (scope vs time)
- Build buffer in estimates
- Communicate risks

**Q208: Hiring senior engineers**
- Look for: Problem-solving, humility, growth mindset
- Technical bar + culture fit
- Involve team in interviews
- Competitive offer, flexibility
- Clear role/impact

**Q209: Build vs buy**
- Build: Control, learning, long-term cost
- Buy: Time-to-market, vendor support, dependencies
- Matrix: Cost, timeline, complexity, risk

**Q210: Cross-team alignment**
- Regular sync meetings
- Shared roadmap
- Clear ownership
- Escalation path
- Psychological safety

**Q211: Incident ownership**
- Declare immediately
- Get SMEs involved
- Communicate status updates
- Focus on resolution, not blame
- Learn from incident

**Q212: Postmortems**
- Blameless culture
- Focus on systems, not people
- Root cause analysis
- Actionable follow-ups
- Share learnings widely

**Q213: Delegation**
- Match task to growth area
- Provide context
- Agree on success criteria
- Available for questions
- Give autonomy

**Q214: Roadmap planning**
- Align with business goals
- Consider technical debt
- Estimate carefully
- Build in buffer
- Communicate trade-offs

**Q215: Team scaling**
- When to hire: Capacity, skills gaps
- Hiring pipeline
- Onboarding process
- Mentoring new hires
- Don't lose culture

**Q216: Engineering metrics**
- Deployment frequency
- Lead time for changes
- Mean time to recovery
- Change failure rate
- NOT: Lines of code, hours worked

**Q217: Security culture**
- Treat as everyone's responsibility
- Regular training
- Security in architecture
- Automated scanning
- Incident response plan

**Q218: Quality gates**
- Automated testing
- Code review requirements
- Performance baselines
- Security scanning
- Staging environment testing

**Q219: Legacy rewrite decisions**
- Cost-benefit analysis
- Timeline risk
- Team capability
- Customer impact
- Strangler fig vs big bang

**Q220: Executive communication**
- Tailor message: Business impact
- Use metrics, not jargon
- Highlight risks early
- Offer solutions
- Regular updates

**Q221: Handling disagreement**
- Listen to understand
- Separate person from idea
- Decide and commit
- Support decision
- Revisit if data changes

**Q222: Career growth for team**
- Individual growth plans
- Stretch assignments
- Learning opportunities
- Promotion transparency
- Mentor next generation

**Q223: Managing distributed teams**
- Async-first mindset
- Clear documentation
- Time zone awareness
- Regular video calls
- Build relationships

**Q224: Cost accountability**
- Track infrastructure costs
- Right-size resources
- Optimize cloud spend
- Educate team on costs
- Balance cost vs performance

**Q225: First 90 days as Tech Lead**
```
Week 1-2:
- Listen more than talk
- Understand team, projects, priorities
- Build relationships
- Learn code, architecture
- Check on health metrics

Week 3-4:
- Identify quick wins
- Understand pain points
- Prioritize improvements
- Build trust through actions

Week 5-8:
- Implement improvements
- Address technical debt
- Clarify processes
- Mentor individuals
- Set direction

Week 9-12:
- Establish patterns
- Build roadmap
- Reflect and adjust
- Set expectations clearly
```
